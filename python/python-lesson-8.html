8.1 Classes and Objects

Classes and objects are fundamental concepts in object-oriented programming (OOP), providing a way to structure and model code. In this lesson, we'll explore the basics of classes, objects, and their usage in Python.

1. Creating a Class:

  • Use the class keyword to create a class. A class encapsulates data (attributes) and behavior (methods).

# Creating a simple class
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f"{self.name} says Woof!")

2. Creating Objects (Instances):

  • Instantiate objects from a class. Each object is a unique instance of that class.

# Creating instances of the Dog class
dog1 = Dog("Buddy", 3)
dog2 = Dog("Max", 5)

3. Accessing Attributes:

  • Access the attributes of an object using dot notation.

# Accessing attributes of objects
print(dog1.name)  # Output: Buddy
print(dog2.age)   # Output: 5

4. Calling Methods:

  • Call methods on objects to perform actions.

# Calling methods of objects
dog1.bark()  # Output: Buddy says Woof!
dog2.bark()  # Output: Max says Woof!

5. Constructors and Initialization:

  • The __init__ method is a constructor that initializes the object's attributes when it is created.

class Cat:
    def __init__(self, name, color):
        self.name = name
        self.color = color

    def meow(self):
        print(f"{self.name} says Meow!")

6. Inheritance:

  • Create a new class that inherits attributes and methods from an existing class.

class GoldenRetriever(Dog):
    def fetch(self):
        print(f"{self.name} is fetching the ball!")

7. Encapsulation:

  • Encapsulation restricts access to certain attributes and methods, controlling the visibility of internal details.

class BankAccount:
    def __init__(self, balance):
        self._balance = balance  # Single underscore indicates a protected attribute

    def get_balance(self):
        return self._balance

8. Polymorphism:

  • Polymorphism allows objects of different classes to be treated as objects of a common base class.

class Animal:
    def make_sound(self):
        pass

class Dog(Animal):
    def make_sound(self):
        print("Woof!")

class Cat(Animal):
    def make_sound(self):
        print("Meow!")

9. Class Attributes:

  • Class attributes are shared among all instances of a class.

class Circle:
    pi = 3.14  # Class attribute

    def __init__(self, radius):
        self.radius = radius

    def calculate_area(self):
        return self.pi * self.radius ** 2

10. Conclusion:

Classes and objects form the basis of object-oriented programming, allowing you to model real-world entities in your code. Understanding how to create and use classes, instantiate objects, and leverage OOP principles such as inheritance and encapsulation is essential for building scalable and maintainable Python programs. Practice designing and working with classes to enhance your object-oriented programming skills.



8.2 Inheritance

Inheritance is a powerful concept in object-oriented programming that allows a new class to inherit attributes and methods from an existing class. In this lesson, we'll delve into the basics of inheritance in Python.

1. Creating a Base Class:

  • Define a base class with common attributes and methods.

class Animal:
    def __init__(self, name):
        self.name = name

    def make_sound(self):
        pass

2. Creating a Derived Class:

  • Create a derived class that inherits from the base class.

class Dog(Animal):
    def make_sound(self):
        return "Woof!"

3. Accessing Base Class Methods:

  • The derived class can access and override methods from the base class.

class Cat(Animal):
    def make_sound(self):
        return "Meow!"

    def purr(self):
        return "Purrrr..."

4. Calling Base Class Method:

  • Use super() to call a method from the base class within the derived class.

class Bird(Animal):
    def make_sound(self):
        return "Chirp!"

    def tweet(self):
        return super().make_sound() + " Tweet!"

5. Overriding vs. Extending:

  • Overriding involves providing a new implementation for a method in the derived class, while extending involves adding new methods to the derived class.

class Fish(Animal):
    def make_sound(self):
        return "Blub!"

    def swim(self):
        return "Swimming gracefully."

6. Multiple Inheritance:

  • A class can inherit from multiple base classes.

class Amphibian(Fish, Frog):
    def make_sound(self):
        return super().make_sound() + " Croak!"

7. Method Resolution Order (MRO):

  • Python follows the C3 Linearization algorithm to determine the method resolution order.

class A:
    def show(self):
        print("A")

class B(A):
    def show(self):
        print("B")

class C(A):
    def show(self):
        print("C")

class D(B, C):
    pass

d = D()
d.show()  # Output: B

8. Abstract Base Classes:

  • Use the ABC module to create abstract base classes with abstract methods.

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

9. Conclusion:

Inheritance is a fundamental concept in object-oriented programming that promotes code reusability and hierarchical structuring of classes. By understanding how to create base and derived classes, override and extend methods, and manage multiple inheritance, you can design more modular and scalable Python programs. Practice implementing inheritance in various scenarios to solidify your understanding of this important OOP principle.



8.3 Polymorphism

Polymorphism is a core concept in object-oriented programming that allows objects of different classes to be treated as objects of a common base class. In this lesson, we'll explore the basics of polymorphism in Python.

1. Creating a Base Class:

  • Define a base class with a method that will be overridden by derived classes.

class Animal:
    def make_sound(self):
        pass

2. Implementing Polymorphism:

  • Create multiple classes that inherit from the base class and implement the same method.

class Dog(Animal):
    def make_sound(self):
        return "Woof!"

class Cat(Animal):
    def make_sound(self):
        return "Meow!"

class Bird(Animal):
    def make_sound(self):
        return "Chirp!"

3. Treating Objects Uniformly:

  • Use polymorphism to treat objects of different classes uniformly.

def animal_sound(animal):
    return animal.make_sound()

dog = Dog()
cat = Cat()
bird = Bird()

print(animal_sound(dog))  # Output: Woof!
print(animal_sound(cat))  # Output: Meow!
print(animal_sound(bird)) # Output: Chirp!

4. Duck Typing:

  • In Python, the type or class of an object is less important than the methods it implements.

class Car:
    def make_sound(self):
        return "Vroom!"

class Guitar:
    def make_sound(self):
        return "Strumming the strings."

def object_sound(obj):
    return obj.make_sound()

car = Car()
guitar = Guitar()

print(object_sound(car))    # Output: Vroom!
print(object_sound(guitar)) # Output: Strumming the strings.

5. Operator Overloading:

  • Implement special methods to allow objects to respond to operators differently.

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

v1 = Vector(1, 2)
v2 = Vector(3, 4)
result = v1 + v2
print(result.x, result.y)  # Output: 4 6

6. Abstract Base Classes (ABCs):

  • Use abstract base classes to define a common interface for multiple classes.

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

7. Conclusion:

Polymorphism allows for flexibility and extensibility in object-oriented programming. By treating objects of different classes uniformly and leveraging concepts like duck typing and operator overloading, you can write code that is more adaptable to changes and promotes code reuse. Practice implementing polymorphism in various scenarios to deepen your understanding of this powerful OOP concept.



8.4 Encapsulation

Encapsulation is a fundamental concept in object-oriented programming that involves bundling data (attributes) and methods that operate on that data within a single unit known as a class. In this lesson, we'll explore the basics of encapsulation in Python.

1. Creating a Class with Encapsulation:

  • Define a class with private attributes and methods.

class BankAccount:
    def __init__(self, account_number, balance):
        self._account_number = account_number  # Protected attribute
        self.__balance = balance  # Private attribute

    def deposit(self, amount):
        self.__balance += amount

    def withdraw(self, amount):
        if amount <= self.__balance:
            self.__balance -= amount
        else:
            print("Insufficient funds.")

    def get_balance(self):
        return self.__balance

2. Access Modifiers:

  • In Python, encapsulation is achieved using access modifiers:

    • Single underscore (_) indicates a protected attribute or method.

    • Double underscore (__) indicates a private attribute or method.

3. Using Encapsulated Class:

  • Instantiate the class and interact with it using its methods.

account = BankAccount("123456", 1000)

account.deposit(500)
account.withdraw(200)
print(account.get_balance())  # Output: 1300

4. Property Decorators:

  • Use property decorators to create read-only or write-only properties.

class Student:
    def __init__(self, name, age):
        self._name = name  # Protected attribute
        self._age = age    # Protected attribute

    @property
    def name(self):
        return self._name

    @property
    def age(self):
        return self._age

    @age.setter
    def age(self, new_age):
        if new_age >= 0:
            self._age = new_age
        else:
            print("Age cannot be negative.")

5. Getter and Setter Methods:

  • Use getter and setter methods for more controlled access to attributes.

class Employee:
    def __init__(self, name, salary):
        self._name = name  # Protected attribute
        self._salary = salary  # Protected attribute

    def get_name(self):
        return self._name

    def get_salary(self):
        return self._salary

    def set_salary(self, new_salary):
        if new_salary >= 0:
            self._salary = new_salary
        else:
            print("Salary cannot be negative.")

6. Benefits of Encapsulation:

  • Data Hiding: Encapsulation hides the internal state of an object, providing only controlled access.

  • Modularity: Encapsulation promotes modularity by bundling related data and methods into a single unit (class).

  • Security: Encapsulation helps in protecting sensitive data by restricting direct access.

7. Conclusion:

Encapsulation is a key principle of object-oriented programming that enhances code organization, security, and maintainability. By encapsulating attributes and methods within a class and controlling access through access modifiers, you can create more robust and modular code. Practice using encapsulation in your classes to improve the design and structure of your Python programs.
