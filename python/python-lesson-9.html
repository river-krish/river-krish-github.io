9.1 Creating Modules

Modules in Python are files containing Python statements and definitions. They allow you to organize code into reusable units, enhancing the maintainability and structure of your projects. In this lesson, we'll explore the basics of creating and using modules in Python.

1. Creating a Simple Module:

  • Create a Python file (e.g., my_module.py) with functions and variables.

# my_module.py

def greet(name):
    return f"Hello, {name}!"

def square(x):
    return x ** 2

pi = 3.1415926535

2. Importing a Module:

  • Import the module in another Python script to access its functions and variables.

# main_script.py

import my_module

print(my_module.greet("Alice"))  # Output: Hello, Alice!
print(my_module.square(5))       # Output: 25
print(my_module.pi)              # Output: 3.1415926535

3. Importing Specific Items:

  • Import specific functions or variables from a module to reduce namespace pollution.

# main_script.py

from my_module import greet, square

print(greet("Bob"))   # Output: Hello, Bob!
print(square(3))       # Output: 9

4. Alias for Modules:

  • Use an alias for a module to provide a shorter name when importing.

# main_script.py

import my_module as mm

print(mm.greet("Charlie"))  # Output: Hello, Charlie!
print(mm.square(4))          # Output: 16
print(mm.pi)                 # Output: 3.1415926535

5. Executing Modules as Scripts:

  • Include a block of code that only runs when the module is executed as a script.

# my_module.py

def greet(name):
    return f"Hello, {name}!"

def square(x):
    return x ** 2

pi = 3.1415926535

if __name__ == "__main__":
    # This code will run if the module is executed as a script
    print(greet("Dave"))
    print(square(2))

6. Module Search Path:

  • Python searches for modules in directories listed in sys.path.

import sys

print(sys.path)

7. Package:

  • A package is a way of organizing related modules into a directory hierarchy.

my_package/
|-- __init__.py
|-- module1.py
|-- module2.py

8. Importing from Packages:

  • Import modules from a package using dot notation.

# main_script.py

from my_package import module1, module2

print(module1.function1())
print(module2.function2())

9. Conclusion:

Creating and using modules in Python is an essential part of code organization and reuse. By encapsulating code into modular units, you can improve the structure and maintainability of your projects. Practice creating modules and packages to enhance your skills in organizing and managing code in Python.





9.2 Importing Modules

Importing modules is a fundamental aspect of Python programming, allowing you to access and use functionalities defined in external files. In this lesson, we'll delve into various ways to import modules and make use of their features.

1. Importing Entire Module:

  • Use the import keyword to import an entire module.

# main_script.py

import my_module

print(my_module.greet("Alice"))
print(my_module.square(5))
print(my_module.pi)

2. Importing Specific Items:

  • Import only specific functions or variables from a module to reduce namespace pollution.

# main_script.py

from my_module import greet, square

print(greet("Bob"))
print(square(3))

3. Importing with Alias:

  • Use an alias for a module or imported items to provide a shorter name.

# main_script.py

import my_module as mm

print(mm.greet("Charlie"))
print(mm.square(4))
print(mm.pi)

4. Importing All Items:

  • Import all items from a module using the * wildcard.

# main_script.py

from my_module import *

print(greet("Dave"))
print(square(2))
print(pi)

5. Importing Specific Items into Current Namespace:

  • Import specific items into the current namespace without the module prefix.

# main_script.py

from my_module import greet as custom_greet

print(custom_greet("Eve"))

6. Importing Modules from Packages:

  • Import modules from packages using dot notation.

# main_script.py

from my_package import module1, module2

print(module1.function1())
print(module2.function2())

7. Importing with __init__.py:

  • The __init__.py file in a package directory indicates that it should be treated as a package.

# __init__.py

# Empty or with initialization code

8. Importing and Executing as Script:

  • Use the if __name__ == "__main__": block in a module to execute code only if the module is run as a script.

# my_module.py

def greet(name):
    return f"Hello, {name}!"

def square(x):
    return x ** 2

pi = 3.1415926535

if __name__ == "__main__":
    print(greet("Dave"))
    print(square(2))

9. Importing External Libraries:

  • Import external libraries/modules that are not part of the Python standard library.

# Using the `requests` library for HTTP requests
import requests

response = requests.get("https://www.example.com")
print(response.status_code)

10. Conclusion:

Understanding different ways to import modules is crucial for building modular and maintainable Python programs. Whether you are importing entire modules, specific items, or using aliases, choose the method that best suits your code organization and readability. Practice importing modules in various scenarios to strengthen your skills in managing dependencies and structuring your code.

9.3 Working with External Packages

External packages, also known as third-party libraries or modules, extend the functionality of Python by providing pre-written code for specific tasks. In this lesson, we'll explore how to work with external packages and leverage their features in your Python projects.

1. Installing External Packages:

  • Use the pip command to install external packages.

pip install package_name

2. Importing External Packages:

  • Import external packages into your Python scripts to use their functionalities.

# Using the `requests` library for HTTP requests
import requests

response = requests.get("https://www.example.com")
print(response.status_code)

3. Virtual Environments:

  • Use virtual environments to manage dependencies for different projects.

# Create a virtual environment
python -m venv myenv

# Activate the virtual environment
# On Windows
myenv\Scripts\activate
# On macOS/Linux
source myenv/bin/activate

4. Installing Packages from Requirements File:

  • Create a requirements.txt file with a list of packages and versions.

requests==2.26.0
numpy==1.21.2

  • Install packages from the requirements file.

pip install -r requirements.txt

5. Common External Packages:

  • Explore and use popular external packages based on your project needs.

    • Requests: HTTP library for making requests.

    pip install requests

    • NumPy: Numerical computing library.

    pip install numpy

    • Pandas: Data manipulation and analysis library.

    pip install pandas

    • Matplotlib: Data visualization library.

    pip install matplotlib

6. Using External Packages:

  • Incorporate external packages into your code for specific tasks.

# Using NumPy for numerical operations
import numpy as np

numbers = [1, 2, 3, 4, 5]
mean = np.mean(numbers)
print(f"Mean: {mean}")

# Using Pandas for data manipulation
import pandas as pd

data = {'Name': ['Alice', 'Bob', 'Charlie'], 'Age': [25, 30, 22]}
df = pd.DataFrame(data)
print(df)

7. Updating and Uninstalling Packages:

  • Keep packages up-to-date or uninstall them when no longer needed.

# Update a package
pip install --upgrade package_name

# Uninstall a package
pip uninstall package_name

8. Documentation and Community Support:

  • Refer to the official documentation and community forums for external packages to understand their features and get help.

    • Requests Documentation

    • NumPy Documentation

    • Pandas Documentation

    • Matplotlib Documentation

9. Conclusion:

Working with external packages is an integral part of Python development, allowing you to leverage a vast ecosystem of pre-built tools and functionalities. Understanding how to install, import, and use external packages enhances your ability to solve diverse problems efficiently. Explore different packages based on your project requirements and continually update your skills as new packages and versions are released.
