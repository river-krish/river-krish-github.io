7.1 Reading from Files

Reading from files is a common task in Python, allowing you to access and process data stored in external files. In this lesson, we'll explore the basics of reading from text files and discuss best practices for handling file operations.

1. Opening a File:

  • Use the open() function to open a file. Specify the file name and the mode ("r" for reading).

# Opening a file in read mode
file_path = "example.txt"
with open(file_path, "r") as file:
    # File operations go here

2. Reading Entire Contents:

  • Use the read() method to read the entire contents of the file as a string.

with open(file_path, "r") as file:
    content = file.read()
    print(content)

3. Reading Line by Line:

  • Use a loop to read the file line by line using the readline() method.

with open(file_path, "r") as file:
    for line in file:
        print(line.strip())  # Strip removes newline characters

4. Reading All Lines into a List:

  • Use the readlines() method to read all lines into a list.

with open(file_path, "r") as file:
    lines = file.readlines()
    for line in lines:
        print(line.strip())

5. Best Practices:

  • Use the with statement (context manager) when working with files. It ensures that the file is properly closed after usage.

# Good Example
with open(file_path, "r") as file:
    content = file.read()
    # Perform operations

# Avoid
file = open(file_path, "r")
content = file.read()
# Perform operations
file.close()  # Explicitly closing the file

6. Checking File Existence:

  • Before attempting to open a file, check if it exists using the os.path.exists() function.

import os

if os.path.exists(file_path):
    with open(file_path, "r") as file:
        content = file.read()
        # Perform operations
else:
    print(f"The file {file_path} does not exist.")

7. Exception Handling:

  • Wrap file operations in a try-except block to handle potential exceptions, such as file not found or permission issues.

try:
    with open(file_path, "r") as file:
        content = file.read()
        # Perform operations
except FileNotFoundError:
    print(f"The file {file_path} does not exist.")
except Exception as e:
    print(f"An error occurred: {e}")

8. Closing the File Explicitly:

  • While using the with statement is recommended, if you choose to open the file without it, ensure to close it explicitly using the close() method.

file = open(file_path, "r")
content = file.read()
# Perform operations
file.close()  # Explicitly closing the file

9. Conclusion:

Reading from files is a fundamental skill in Python programming. By following best practices, such as using the with statement, checking file existence, and handling exceptions, you can ensure your file operations are robust and error-resistant. Practice reading from files with different scenarios to enhance your file-handling skills.



7.2 Writing to Files

Writing to files is a crucial aspect of handling data in Python, allowing you to store information for later use or share it with others. In this lesson, we'll explore the basics of writing to text files and discuss best practices for file-writing operations.

1. Opening a File for Writing:

  • Use the open() function to open a file in write mode ("w"). If the file does not exist, it will be created. If it already exists, its content will be overwritten.

# Opening a file in write mode
file_path = "output.txt"
with open(file_path, "w") as file:
    # File-writing operations go here

2. Writing a Single Line:

  • Use the write() method to write a single line to the file. Remember to include newline characters ("\n") if you want to separate lines.

with open(file_path, "w") as file:
    file.write("This is a line written to the file.\n")

3. Writing Multiple Lines:

  • You can write multiple lines to the file by calling write() multiple times or using a list of strings with writelines().

with open(file_path, "w") as file:
    lines = ["Line 1\n", "Line 2\n", "Line 3\n"]
    file.writelines(lines)

4. Best Practices:

  • Use the with statement (context manager) to ensure that the file is properly closed after writing.

# Good Example
with open(file_path, "w") as file:
    file.write("This is a line written to the file.\n")
    # Perform operations

# Avoid
file = open(file_path, "w")
file.write("This is a line written to the file.\n")
# Perform operations
file.close()  # Explicitly closing the file

5. Appending to a File:

  • To append content to an existing file, open it in append mode ("a").

with open(file_path, "a") as file:
    file.write("This line will be appended to the file.\n")

6. Exception Handling:

  • Wrap file-writing operations in a try-except block to handle potential exceptions, such as permission issues or disk space problems.

try:
    with open(file_path, "w") as file:
        file.write("This is a line written to the file.\n")
        # Perform operations
except PermissionError:
    print("Permission error: Unable to write to the file.")
except Exception as e:
    print(f"An error occurred: {e}")

7. Using Context Managers:

  • Context managers are the preferred way to work with files in Python, ensuring proper resource management.

with open(file_path, "w") as file:
    file.write("This is a line written to the file.\n")
    # Perform operations

8. Conclusion:

Writing to files is a fundamental skill in Python programming. By adhering to best practices, such as using the with statement, handling exceptions, and appending when necessary, you can ensure robust and reliable file-writing operations. Practice writing to files with different scenarios to enhance your file-handling skills.



7.3 Handling Exceptions

Handling exceptions is a crucial aspect of writing robust and error-resistant Python code. In this lesson, we'll explore the basics of exception handling, including try-except blocks, raising exceptions, and best practices for effectively managing errors in your programs.

1. Try-Except Blocks:

  • Use a try-except block to catch and handle exceptions that might occur during the execution of your code.

try:
    # Code that may raise an exception
    result = 10 / 0
except ZeroDivisionError:
    print("Error: Division by zero!")

2. Multiple Except Blocks:

  • You can have multiple except blocks to handle different types of exceptions.

try:
    # Code that may raise an exception
    result = int("not_an_integer")
except ValueError:
    print("Error: Not a valid integer.")
except ZeroDivisionError:
    print("Error: Division by zero!")

3. Handling Multiple Exceptions:

  • Use a tuple to catch multiple exceptions in a single except block.

try:
    # Code that may raise an exception
    result = int("not_an_integer")
except (ValueError, TypeError):
    print("Error: Not a valid integer.")

4. Else Block:

  • The else block contains code that should be executed if no exceptions are raised.

try:
    # Code that may raise an exception
    result = 10 / 2
except ZeroDivisionError:
    print("Error: Division by zero!")
else:
    print("No error occurred. Result:", result)

5. Finally Block:

  • The finally block contains code that will be executed no matter what, whether an exception is raised or not.

try:
    # Code that may raise an exception
    result = 10 / 2
except ZeroDivisionError:
    print("Error: Division by zero!")
else:
    print("No error occurred. Result:", result)
finally:
    print("This code always runs.")

6. Raising Exceptions:

  • Use the raise statement to manually raise exceptions when certain conditions are met.

try:
    age = -5
    if age < 0:
        raise ValueError("Age cannot be negative.")
except ValueError as ve:
    print(f"Error: {ve}")

7. Custom Exceptions:

  • You can create your own custom exceptions by subclassing the Exception class.

class CustomError(Exception):
    pass

try:
    raise CustomError("This is a custom exception.")
except CustomError as ce:
    print(f"Custom Error: {ce}")

8. Best Practices:

  • Be specific in handling exceptions. Catch only the exceptions you expect and handle them appropriately.

  • Provide informative error messages to aid in debugging.

  • Avoid using a broad except clause that catches all exceptions unless necessary.

# Good Example
try:
    # Code that may raise a specific exception
except SpecificError as se:
    print(f"Error: {se}")
except AnotherSpecificError as ase:
    print(f"Error: {ase}")

# Avoid
try:
    # Code that may raise any exception
except Exception as e:
    print(f"Error: {e}")

9. Conclusion:

Effective exception handling is vital for writing reliable and maintainable Python code. By using try-except blocks, raising exceptions judiciously, and following best practices, you can ensure that your programs gracefully handle errors and provide meaningful feedback. Practice handling exceptions in different scenarios to strengthen your error-handling skills.
